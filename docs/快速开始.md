## 一、准备工作

开始之前，确认你手上有以下两样东西：

| 需要的文件 | 说明 |
|-----------|------|
| 项目文件夹 | 解压后的完整项目目录，里面包含 `scripts/`、`fonts/`、`input/` 等子文件夹 |
| `.env` 文件 | 管理员单独发给你的密钥文件，包含所有 API Key |

**`.env` 文件放在哪里？**

放到项目文件夹的**根目录**，和 `scripts/` 文件夹同一层级，如下所示：

```
skills-store-operational-banner-design/
├── .env                ← 放这里
├── scripts/
├── fonts/
├── input/
└── output/
```

> `.env` 文件包含密钥，不要转发给其他人，不要上传到任何网盘或代码托管平台。

## 二、安装 Python

本项目需要 Python 3.10 或更高版本。

### Windows

1. 打开浏览器，访问 https://www.python.org/downloads/
2. 点击页面上最大的黄色"Download Python 3.x.x"按钮，下载安装包
3. 运行下载好的安装包
4. **关键步骤**：在安装界面最底部，勾选 **"Add Python to PATH"**（这一步很容易漏掉，一定要勾上）

   ```
   ☑ Add Python to PATH    ← 必须勾选
   ```

5. 点击"Install Now"，等待安装完成

**验证安装是否成功：**

按 `Win + R`，输入 `cmd`，回车，打开命令提示符，输入：

```
py --version
```

如果看到类似 `Python 3.10.x` 的输出，说明安装成功。

> Windows 注意：请始终使用 `py` 命令，不要用 `python`。Windows 上的 `python` 命令可能指向一个无效的系统存根程序，会导致脚本没有任何输出、静默失败。

---

### macOS

**方式 A（推荐）：使用官网安装包**

1. 打开浏览器，访问 https://www.python.org/downloads/
2. 下载 macOS 版安装包（`.pkg` 文件）
3. 双击安装，按提示完成

**方式 B：使用 Homebrew**

如果你已经安装了 Homebrew，在终端运行：

```bash
brew install python@3.10
```

**验证安装是否成功：**

打开"终端"（Terminal），输入：

```bash
python3 --version
```

如果看到类似 `Python 3.10.x` 的输出，说明安装成功。

## 三、安装项目依赖

### 打开终端并进入项目目录

**Windows：**

在项目文件夹的空白处，按住 `Shift` 键再点击鼠标右键，选择"在此处打开 PowerShell 窗口"（或"在终端中打开"）。

**macOS：**

打开"终端"（Terminal），把项目文件夹从 Finder 拖进终端窗口，会自动填入路径，然后在路径前加 `cd `（cd 加空格），按回车。

例如：
```bash
cd /Users/你的用户名/Desktop/skills-store-operational-banner-design
```

---

### 运行安装命令

**Windows（PowerShell 里）：**

```
py -m pip install -e .
```

**macOS（终端里）：**

```bash
pip3 install -e .
```

安装过程会下载十几个依赖包，网络正常的情况下约需 1～3 分钟。

**安装成功的标志：**

最后几行出现类似以下内容：

```
Successfully installed Pillow-10.x.x requests-2.x.x python-dotenv-1.x.x ...
```

或者：

```
Successfully installed skills-store-operational-banner-design-0.1.0
```

如果出现红色的 `ERROR`，请截图并联系管理员。

## 四、安装 BiRefNet 抠图模型

BiRefNet 用于专题长图的顶部主体抠图功能。安装分两步：先装依赖包，首次运行时自动下载模型权重（约 1GB）。

### 安装依赖包

**Windows：**

在项目目录下，双击运行：

```
scripts\install_birefnet_deps.bat
```

或者在 PowerShell 里运行：

```
py -m pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
py -m pip install "transformers>=4.38.0,<5.0.0" huggingface_hub "regex>=2024.5.0"
```

**macOS：**

在终端里运行：

```bash
pip3 install -e ".[birefnet]"
```

### 关于模型权重的自动下载

BiRefNet 模型权重**不包含在项目文件夹里**，第一次运行涉及抠图的功能时，程序会自动从网上下载（约 1GB）。

下载时终端会显示进度条，**这不是卡死，请耐心等待**，视网络情况需要 5～30 分钟。

**如果下载很慢或失败：**

在终端设置国内镜像源后再运行脚本（每次打开新终端都需要重新设置）：

Windows PowerShell：
```
$env:HF_ENDPOINT="https://hf-mirror.com"
```

macOS 终端：
```bash
export HF_ENDPOINT=https://hf-mirror.com
```

## 五、配置 API Key

### 放置 .env 文件

将管理员发给你的 `.env` 文件放到项目根目录（参见第一节的目录结构示意图）。

`.env` 是一个普通文本文件，里面存放了访问各 AI 图像 API 的密钥，格式类似：

```
MOXINGPT_API_KEY=sk-xxxxxxxxxxxxxxxx
MOXINGEMINI_API_KEY=sk-xxxxxxxxxxxxxxxx
```

### Windows 特别注意：系统环境变量冲突

Windows 系统中，如果之前手动设置过名为 `GEMINI_API_KEY` 的系统环境变量，它会**覆盖 `.env` 文件中的同名设置**，导致 API 报 401 错误。

**检查方法：**

在 PowerShell 里运行：

```
[Environment]::GetEnvironmentVariable('GEMINI_API_KEY', 'User')
```

如果有输出（显示一串 Key），说明存在冲突，需要清除：

```
[Environment]::SetEnvironmentVariable('GEMINI_API_KEY', $null, 'User')
```

清除后重新打开终端再运行脚本。

## 六、字体安装（macOS 专项）

叠字功能（在 Banner 上合成主标题、副标题）依赖**微软雅黑字体**。Windows 系统自带这个字体，macOS 系统没有，需要手动安装。

### 获取字体文件

管理员会随项目一起提供以下两个字体文件（应已放在项目的 `fonts/` 目录里）：

```
fonts/
├── msyh.ttc       ← 微软雅黑 常规体
└── msyhbd.ttc     ← 微软雅黑 粗体
```

如果 `fonts/` 目录里没有这两个文件，联系管理员补发。

### 安装方法

字体文件放在项目 `fonts/` 目录里即可，**不需要安装到系统**，脚本会优先从这里读取。

**验证字体是否到位：**

在终端里运行（进入项目目录后）：

```bash
ls fonts/
```

确认输出中包含 `msyh.ttc` 和 `msyhbd.ttc`。

## 七、第一次运行测试

所有安装步骤完成后，运行以下命令验证环境是否正常，这条命令**不会调用任何 API，不会消耗费用**：

**Windows：**

```
py scripts/ensure_python.py
```

**macOS：**

```bash
python3 scripts/ensure_python.py
```

**预期输出（正常）：**

```
✅ 找到 Python: /usr/local/bin/python3
   版本: Python 3.10.x
✅ Python 版本满足要求（3.8+）
```

如果看到 `✅` 标志，说明 Python 环境配置正确，可以正常使用了。

## 八、日常使用方法

### 基本命令格式

**Windows：**

```
py scripts/run_full_with_custom_prompt.py -g "分组名" -m "主标题" -s "副标题"
```

**macOS：**

```bash
python3 scripts/run_full_with_custom_prompt.py -g "分组名" -m "主标题" -s "副标题"
```

### 常用分组名

| 分组名 | 包含广告位数量 | 用途 |
|--------|:---:|------|
| `商店日常` | 8 个 | PC 商店全套（首页横幅、专题封面、头图、长图等） |
| `商店移动端日常` | 6 个 | 移动端商店全套（Banner、专题封面、通知等） |
| `LZ全部` | 13 个 | Legion Zone 全套广告位 |

### 示例命令

生成商店日常全套（8 个尺寸）：

**Windows：**
```
py scripts/run_full_with_custom_prompt.py -g "商店日常" -m "开学回血补给站" -s "升级学习力"
```

**macOS：**
```bash
python3 scripts/run_full_with_custom_prompt.py -g "商店日常" -m "开学回血补给站" -s "升级学习力"
```

### 查看输出结果

生成完成后，结果保存在项目根目录的 `output/` 文件夹里，子文件夹名称包含分组名和主标题，例如：

```
output/
└── 商店日常_开学回血补给站_20260717_143022/
    ├── 首页 1976x464.png
    ├── 专题封面 500x280.png
    ├── 商店田字格 304x216.png
    └── ...
```

直接打开 `output/` 文件夹查看即可。

## 九、常见问题

### `py` 命令找不到 / 不是内部命令（Windows）

**原因：** Python 安装时没有勾选"Add Python to PATH"。

**解决：** 重新运行 Python 安装包，选择"Modify"→勾选"Add Python to PATH"→确认。或者卸载后重新安装，这次记得勾选。

---

### API 报 401 Unauthorized 错误

**原因 1：** `.env` 文件没有放在正确位置。确认 `.env` 和 `scripts/` 文件夹在同一层级（项目根目录）。

**原因 2（Windows 专属）：** 系统环境变量里有同名变量覆盖了 `.env`。按第五节的方法检查并清除。

---

### 叠字显示方块或乱码（macOS）

**原因：** `fonts/` 目录里缺少微软雅黑字体文件。

**解决：** 按第六节的说明，确认 `fonts/msyh.ttc` 和 `fonts/msyhbd.ttc` 存在，如果没有联系管理员补发。

---

### BiRefNet 模型下载很慢或中断

**解决：** 设置国内镜像后重新运行脚本（终端不要关，在同一窗口里执行）：

Windows：
```
$env:HF_ENDPOINT="https://hf-mirror.com"
```

macOS：
```bash
export HF_ENDPOINT=https://hf-mirror.com
```

设置后直接重新运行之前的脚本，下载会从失败处续传。

---

### 脚本运行后没有任何输出（Windows）

**原因：** 使用了 `python` 命令而不是 `py`。Windows 上 `python` 可能指向 Microsoft Store 的无效存根。

**解决：** 把命令里所有的 `python` 改成 `py`。

---

### 提示"未找到可用的 Python 解释器"

**解决：** 运行 `py scripts/ensure_python.py`（Windows）或 `python3 scripts/ensure_python.py`（macOS），根据提示信息排查。通常是 Python 未正确安装或未加入 PATH。

---

### 生成过程报网络错误 / 连接超时

**原因：** API 服务器暂时不可用，或本机网络有代理/防火墙限制。

**解决：** 
1. 稍等几分钟后重试
2. 如果长期失败，联系管理员确认当前可用的 API 后端

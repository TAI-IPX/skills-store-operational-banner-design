## 1. 快速上手（5 分钟）

### 最小可运行命令

```bash
# 方案 A：给标题，AI 自动生成背景图 + 叠字
py scripts/run_full_with_custom_prompt.py -g 商店日常 --moxingpt --moxingemini \
  -m "你的主标题" -s "你的副标题"

# 方案 B：给图片，自动处理背景 + 叠字
py scripts/run_all_presets.py input/uploads/current.png \
  -m "你的主标题" -s "你的副标题" -g 商店日常 --moxingpt --moxingemini
```

### 运行步骤

1. `cd skills-store-operational-banner-design`（必须在项目根目录）
2. 确认 `.env` 已配置 API Key（见下方后端配置）
3. 粘贴图片到对话框（方案 B）或准备描述文字（方案 A）
4. 运行命令
5. 输出在 `output/` 目录

## 2. 后端配置速查

### 推荐组合（日常使用）

| 组合 | 生图后端 | 编辑后端 | 适用场景 |
|------|---------|---------|---------|
| **推荐** | `--moxingpt` | `--moxingemini` | 日常首选，稳定快速 |
| 仅 Gemini | 不填 | `--moxingemini` | 仅编辑已有图片 |

### .env 最小配置

```ini
# 必须配置（编辑/扩图/Vision）
MOXINGEMINI_API_KEY=sk-your_moxingemini_token

# 必须配置（生图）
MOXINGPT_API_KEY=sk-your_moxingpt_token
```

### 11 个后端 Key 一览

| Key 变量 | 对应指令 | 用途 | 是否必要 |
|----------|---------|------|---------|
| `MOXINGPT_API_KEY` | `--moxingpt` | gpt-image-2 生图（当前使用） | **是** |
| `MOXINGEMINI_API_KEY` | `--moxingemini` | Gemini 编辑/Vision（当前使用） | **是** |
| `PACKYGPT_API_KEY` | `--packygpt` | gpt-image-2 生图（备选） | 备选 |
| `PACKY7S_API_KEY` | `--packy7s` | Gemini 编辑/Vision（备选） | 备选 |
| `MICUAPI_API_KEY` | `--micugpt2` | gpt-image-2 生图（备选） | 备选 |
| `GEMINI_API_KEY` | 默认 | Gemini 编辑 | 备选 |
| `PACKY_API_KEY` | `--packy` | Gemini 编辑（老版 packy） | 几乎不用 |
| `PACKY3S_API_KEY` | `--packy3s` | Gemini 编辑（第三代 packy） | 几乎不用 |
| `XINGCHENGGPT_API_KEY` | `--xingchengpt` | gpt-image-2 生图（星程） | 几乎不用 |
| `XINCHENGPT_API_KEY` | `--xinchengpt` | gpt-image-2 生图（新城） | 几乎不用 |
| `XINGCHENGEMINI_API_KEY` | `--xingchengemini` | Gemini 编辑（星程） | 几乎不用 |

> 几乎不用的 Key 不需要配置，除非有特殊需求或原来的 Key 失效需要替换。

## 3. 指令分类参考

### 3.1 必要指令（5 个）

这 5 个是你**每次运行都必须用到**的指令：

| 指令 | 短写 | 说明 | 示例 |
|------|------|------|------|
| `--main-title` | `-m` | 主标题文字 | `-m "开学季特惠"` |
| `--subtitle` | `-s` | 副标题文字 | `-s "全场五折起"` |
| `--group` / `--genre` | `-g` | 场景分组（决定走哪条管线） | `-g 商店日常` |
| `--image`（位置参数） | `-i` | 输入图片路径（方案 B 必填） | `-i input/uploads/current.png` |
| `--output-dir` | `-o` | 输出目录（不填自动创建） | `-o output/my_banner` |

**`-g` 可选值速查**：

| 分组名 | 用途 | 输出尺寸 |
|--------|------|---------|
| `商店日常` | 首页/专题/题头等常规 Banner | 8 个预设（1976×464 等） |
| `商店移动端日常` | 移动端 Banner | 3 个预设（984×442 等） |
| `开放平台` | 开放平台 Banner | 2 个预设 |
| `活动长图` | 竖版活动海报 | 1 个预设（changtu_poster） |
| `战报` | 数据战报长图 | 1 个预设（1080px 竖版） |
| `排行榜` | 排行榜长图 | 1 个预设（1080px 竖版） |
| `邮件长图` | 邮件竖版长图 | 1 个预设（1920px 竖版） |
| `商店畅玩卡1920*550` | 畅玩卡 Banner | 2 个预设 |
| `商店题头图 1740*220` | 题头条 Banner | 1 个预设 |
| `商店田字格 304*216` | 田字格小图 | 1 个预设 |
| `商店首页 1976*464` | 首页大 Banner | 1 个预设 |

## 5. 注意事项

### 必须遵守

| 事项 | 说明 |
|------|------|
| **用 `py` 不用 `python`** | Windows 的 `python` 可能指向 Microsoft Store 存根，不会实际执行 |
| **在项目根目录运行** | `cd skills-store-operational-banner-design` 后再运行命令 |
| **图片粘贴后直接跑脚本** | 粘贴图片到对话框时模型报错是正常的，忽略即可，脚本会自动提取 |
| **`.env` 不要提交** | API Key 在 `.env` 中，已在 `.gitignore` 中排除 |

## 6. 故障排查

| 问题 | 解决方案 |
|------|---------|
| `python` 命令无输出 | 改用 `py` 前缀 |
| 401 错误 | API Key 无效或过期，检查 `.env` |
| 403 错误 | Key 对该模型无权，检查 Key 对应的后端 |
| 503 错误 | 服务端临时不可用，稍后重试或换后端 |
| 图片没提取到 | 手动用 `-i` 指定路径，或运行 `py scripts/set_upload_image.py 图片路径 --copy` |
| 生图走了错误后端 | 检查命令行参数，确认后端组合正确 |
| 主体溢出画面 | 图片中主体太靠边，尝试换一张图或调整 bbox |

# 图片处理说明

## 问题背景

当前使用的 deepseek 模型不支持图片输入。粘贴图片到对话框时，模型会报错：
```
ERROR: Cannot read "xxx.png" (this model does not support image input).
```

## 实际流程（不影响运行）

图片粘贴后，OpenCode 会将图片数据传入对话流。**即使模型不支持图片输入而报错，JS 插件和后续逻辑也不受影响**。

项目内所有支持图片的脚本，已接入自动提取流程：

```
粘贴图片 → 模型报错（但图片仍传入 JS 插件）
                ↓
      JS 插件 image-saver.js 自动解码保存
                ↓
      input/uploads/current.png + input/upload_path.txt
                ↓
      脚本调用 opencode_image_input.extract_latest()
          （作为无插件环境的补充回退）
                ↓
      正常执行（抠图、合成、叠字）
```

## 使用方式

上传图片后，直接跑脚本，**无需任何额外操作**：

```bash
# 商店田字格
py scripts/run_shop_mobile_tianzige.py -m "标题" -c 蓝色

# 全预设 Banner
py scripts/run_all_presets.py -g "商店日常" -m "标题" -s "副标题"

# 专题长图
py scripts/run_wide_only.py -m "标题" -s "副标题"
```

## 如果提取失败

脚本因找不到图片而报错时，手动尝试：

```bash
py lib/opencode_image_input.py
```

查看是否有输出。若仍失败，说明图片未成功存入 DB，请重新粘贴图片后重试。

## 脚本间关系

| 文件 | 角色 |
|------|------|
| `lib/opencode_image_input.py` | 统一提取模块，DB → 缓存 → 已有文件 |
| `lib/image-saver.js` | JS 插件副本，供复制到新项目 |
| `lib/image-saver.config.json` | JS 插件配置模板 |
| `.opencode/plugins/image-saver.js` | JS 插件（本机实际使用） |
| `_paths.auto_extract_latest()` | 共享函数，入口脚本自动调用 |
| `input/uploads/current.png` | 最新图片固定路径 |
| `input/upload_path.txt` | 路径指针，入口脚本依赖 |
| `grab_db_image.py` | 兼容存根，委托给 `opencode_image_input` |
| `grab_opencode_image.py` | 兼容存根，供旧代码导入 |

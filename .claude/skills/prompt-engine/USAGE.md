# PROMPT_SYSTEM 复用指南

> 本文件说明如何在其他项目中复用 `PROMPT_SYSTEM.md` 生成中文 AI 绘图 Prompt。

---

## 一、所需依赖

| 组件 | 说明 | 用途 |
|------|------|------|
| **LLM 模型**（任意） | 支持长上下文（≥ 8K tokens）的文本模型 | 运行 PROMPT_SYSTEM.md 生成 Prompt |
| **生图 API / 模型** | Midjourney / DALL-E / SD / gpt-image-2 / 即梦 等 | 接收中文 Prompt 生成图片 |
| **PROMPT_SYSTEM.md** | 本项目的 System Prompt 文件 | 传给 LLM 作为生成指令 |

**链路：** `PROMPT_SYSTEM.md → LLM → 中文 Prompt → 生图工具 → 图片`

---

## 二、模型选择

### 2.1 推荐模型（按效果排序）

| 推荐等级 | 模型 | 上下文窗口 | 推荐理由 |
|---------|------|---------|---------|
| ★★★★★ | Claude 4 / 3.5 Sonnet | 200K | 指令遵循最佳，输出长度控制好，不偷懒 |
| ★★★★ | GPT-4o / GPT-4.1 | 128K | 推理强，中文流畅，偶尔过度发挥 |
| ★★★★ | DeepSeek V3 / R1 | 128K | 中文优秀，成本低，逻辑严谨 |
| ★★★ | GPT-4o-mini / Claude 3 Haiku | 128K / 200K | 成本低但有概率偷懒、跳过推导步骤 |
| ★★ | DeepSeek V2.5 及以下 | 128K | 可能无法稳定完成全部 6 步管道 |
| ★ | 任意 < 8K 上下文的模型 | — | **不推荐**：PROMPT_SYSTEM.md 本身约 5K tokens，加上对话余量不够 |

### 2.2 各模型使用特点

| 模型 | 优势 | 注意事项 |
|------|------|---------|
| Claude 4 Sonnet | 不偷懒，逐步骤执行 | 偶尔过度谨慎 |
| GPT-4o | 推理强，理解用户意图好 | 可能跳过「信息解析卡」直接出 Prompt |
| DeepSeek V3 | 中文最强，成本最低 | 默认行为偏简洁，需强调「严格按 6 步执行」 |
| GPT-4o-mini | 速度快，适合批量 | 有概率生成英文 Prompt（违反语言约束） |

### 2.3 推荐 Prompt 写法

在 PROMPT_SYSTEM.md 之后附上用户输入时，加上这句以强化指令遵循：

```
请严格按上述文档的 6 步管道执行，每一步都必须输出对应的分析卡片，不允许跳过任何步骤。
```

## 三、集成方式

### 方式 A：ChatGPT / Claude 对话（最简单）

1. 打开 ChatGPT / Claude 网页版
2. 将 `PROMPT_SYSTEM.md` 全文粘贴为第一条消息
3. 第二条消息发送用户输入：
   ```
   -m "科研必备绘图工具" -s "做好图表 秒发顶刊"
   ```
4. LLM 输出中文 Prompt 后，复制到生图工具生成图片

### 方式 B：API 调用（Python）

```python
# 以 OpenAI API 为例
from openai import OpenAI

client = OpenAI(api_key="your-key")

# 读取 System Prompt
with open("PROMPT_SYSTEM.md", "r", encoding="utf-8") as f:
    system_prompt = f.read()

user_input = '-m "科研必备绘图工具" -s "做好图表 秒发顶刊"'

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_input}
    ],
    temperature=0.7,
    max_tokens=4096
)

print(response.choices[0].message.content)
```

### 方式 C：OpenCode / Claude Code / 自定义 Agent

在你的 Agent 系统 Prompt 中前置加载 PROMPT_SYSTEM.md：

```markdown
## 角色
你是一个 Prompt 生成 Agent。

## 核心指令
{PROMPT_SYSTEM.md 全文内容}

## 用户输入
[用户的实际输入]
```

## 四、输入格式

### 4.1 基本输入

```
-m "主标题" -s "副标题"
```

```
我想做一张小满节气的海报
```

### 4.2 可选覆盖指令

```
-m "摸鱼作战计划" -s "一直摸鱼一直爽！"
含文字版                               # 激活标题/副标题排版
风格=CG厚涂  色调=暖棕色调               # 覆盖视觉风格和色调
构图=宏大仰视式  光线=丁达尔效应           # 覆盖构图和光线
品牌层级=高端                           # 品牌层级注入
```

### 4.3 参数速查

| 参数 | 说明 | 示例 |
|------|------|------|
| `-m` | 主标题（核心主题） | `-m "摸鱼作战计划"` |
| `-s` | 副标题（补充说明） | `-s "一直摸鱼一直爽！"` |
| `风格=N` | 强制指定风格 1-16 | `风格=6`（CG厚涂） |
| `色调=描述` | 覆盖色调 | `色调=暖棕色调` |
| `构图=描述` | 覆盖构图类型 | `构图=宏大仰视式` |
| `光线=描述` | 覆盖光线 | `光线=丁达尔效应` |
| `主体=描述` | 跳过推导，直接指定主体 | `主体=一台显微镜` |
| `品牌层级=描述` | 覆盖品牌层级 | `品牌层级=高端` |
| `mood=plain` | 退回克制风格模式（默认运营活动高张力） | `mood=plain` |
| `mood=oppressive` | 允许纯震撼向，跳过情感基线（默认禁用） | `mood=oppressive` |
| `不改prompt` | 重复输入时复用上次 Prompt | `不改prompt` |
| `person=realistic` | 解禁写实人像（默认禁用） | `person=realistic` |
| `含文字版` | 激活排版模板 | 输出标题/副标题/CTA |

## 五、常见问题

### Q1: 生成的 Prompt 图上有文字，怎么去掉？

默认模式即为无文字版。如果图上仍出现标题/副标题，检查是否误加了「含文字版」参数。

场景内的装饰性文字（便签、涂鸦、气泡等）**不受影响，这是正常的**。

### Q2: 生成的 Prompt 太抽象，图看起来不真实？

检查 `PROMPT_SYSTEM.md` 中的「具象锚点」规则是否生效（原则 9）。如果 LLM 仍输出全息投影/粒子流等抽象画面，是模型未严格遵循指令——换 Claude 4 或 GPT-4o 通常能解决。

### Q3: LLM 跳过某些步骤不输出分析卡片？

在用户输入末尾追加：
```
请严格按照 6 步管道执行，每一步都输出分析卡片，不要跳过。
```

### Q4: 模型输出了英文 Prompt？

中文约束在 PROMPT_SYSTEM.md 原则 2 中声明。如果模型违反，是该模型的多语言指令遵循能力不足——换 Claude 4 或 DeepSeek V3 可解决。

### Q5: 每次生成风格一样怎么办？

系统内置了索引轮转机制（第十章），每次生成后风格/构图/主体变体自动 +1，确保跨次视觉多样性。如果连续多次生成在同一对话中，效果最明显。

### Q6: 默认就是运营活动高张力，需要克制风格怎么办？

附加 `mood=plain` 参数即可退回克制风格模式：

```
-m "小满节气" -s "麦穗初满" mood=plain
```

效果：
- 风格索引回退到完整 16 风格矩阵（启用 Moebius / 羊毛毡 / 壁画质感 / William Morris / 现代艺术 / Armin Hofmann）
- 写作风格从 B（堆叠轰炸式）回退到 A（自然叙述式）
- 末尾「运营活动张力强化段」自动跳过
- Step 5 维度 10/11（张力/氛围检测）自动跳过
- Step 6 维度 13/14（张力强度/氛围饱和度）总分回退到 12 维 / 60 分，阈值 43

不附加 `mood=plain` 时，所有产出默认走运营活动主线（强张力、强氛围）。

### Q7: 重复输入相同主副标题为什么没复用上次的 prompt？

**默认就是「重新生成以保证多样性」**。如果想强制复用上次的 Prompt（不调用 LLM，节省 token），需要显式附加 `不改prompt`：

```
-m "聚力高考护航" -s "多元工具助力备考" 不改prompt
```

效果：
- 命中缓存 → 直接返回上次结果，跳过 6 步管道
- 未命中 → 与普通流程一样生成并写入缓存

缓存文件位置：`output/cache/prompt_engine/{hash}.json`，跨 session 持久化。清理方式：

```bash
py .claude/skills/prompt-engine/scripts/prompt_engine_optimizer.py --clear-cache
```

### Q8: 想要真人模特/写实人像怎么办？以及可以出现其它风格人物吗？

**默认禁用真人/写实人像**（入口级原则 12），但允许其它风格人物。

- **想用真人/写实人像**：附加 `person=realistic`
  ```
  -m "新品发布" -s "高端质感" person=realistic
  ```
  仅对风格 1（好莱坞科幻大片）/ 5（东方玄幻）/ 16（超现实主义未来科幻）有效；其他规则（无 logo/无 IP）仍生效。

- **想要插画/CG/油画/羊毛毡/3D 卡通/水墨/剪影等其它风格人物**：直接用 `风格=N` 选库中风格即可（库中 16 种风格天然非真人）
  ```
  -m "新品发布" -s "高端质感" 风格=8  # 羊毛毡人偶
  -m "新品发布" -s "高端质感" 风格=3  # 3D 立体卡通
  -m "新品发布" -s "高端质感" 风格=6  # CG 厚涂
  ```

### Q9: 想要纯震撼向的画面（不要积极向上收束），比如反派/末日/深渊主题怎么办？

默认走「情感基线」会自动检测主副标题语义，把画面收束在「积极向上」方向（参见入口级原则 13 / Step 1 信息解析卡的「情感方向」字段）。如果主题本身就需要纯震撼、压迫、黑暗、绝望的氛围（如反派登场、末日世界、深渊体验），需要显式附加 `mood=oppressive` 跳过情感基线：

```
-m "末世废土" -s "文明的最后一道防线" mood=oppressive
```

效果：
- Step 4 末尾「情感基调段」自动跳过（不注入"希望/激励/突破/势不可挡"等正向词）
- Step 5 维度 13「情感基线检测」自动跳过（不会自动替换"压迫/悲壮/渺小/失败/绝望"等持续性负面词）
- Step 6 维度 16「情感基线」自动跳过
- 质检总分回退到 15 维 / 75 分，阈值 52
- 与 `person=realistic` 叠加时，质检总分回退到 14 维 / 70 分，阈值 48（跳过维度 15、16）

`mood=oppressive` 是「情感基线」专用覆写开关，不影响运营活动主线（强张力/强氛围）和写实禁令的其它规则。

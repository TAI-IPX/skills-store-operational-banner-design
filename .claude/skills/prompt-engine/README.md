# App Store Cover Generator

输入 App 信息，自动生成应用商店封面展示图。

## 它能做什么

只需提供 App 名称、分类和一句话介绍，自动完成：

- **信息补全** — 信息不足时自主推断视觉方向，不追问
- **风格匹配** — 内置 15 种视觉风格，自动按规则轮换/跳过
- **主体推导** — 优先从 S_INTRO 提取2个具体可视化词，其次根据 NAME 分类提取 HUMAN_DESC，最后 SUBJECT_POOL 兜底
- **品牌色提取** — 自动从 ICON 量化提取占比最高的品牌主色块并提升为高饱和高明度；仅当主色落在正红（0-15°/345-360°）或蓝色（195-260°）高频区间时，根据包名做确定性色相偏移（±45°），橙色、黄色、绿色、青色、紫色等其他色相保持原主色方向
- **质量自检** — 自动评分，不达标自动修正
- **生图输出** — 调用配置的 API 直接出图，压缩至 ≤200KB

---

## 品牌主色全局规则

- **禁止全图平均色作为最终主色**：先过滤透明、近白、近黑像素，再做颜色量化，取占比最高且非中性的品牌色块作为主色。
- **先保品牌识别，再做色彩分散**：主色统一提升到高饱和、高明度，避免灰暗褪色，但不随意改变品牌色相。
- **只偏移正红/蓝色高频区间**：仅当原始主色色相落在正红 `0-15° / 345-360°` 或蓝色 `195-260°` 时，才按包名做稳定色相偏移（±45°）。
- **橙色保护**：橙色约 `15-40°` 不属于红色偏移区间，必须保持橙色品牌方向，避免滴滴这类橙色 Logo 被误推成红色。
- **其他色相不偏移**：黄色、绿色、青色、紫色、玫红等本来有识别度的品牌色保持原始主色方向。

## 风格锁定规则

- 当前 15 个风格、风格专项规则、风格禁用规则和核心构图已锁定在代码签名中。
- 运行 `python3 scripts/batch_generate.py` 时会先执行风格锁校验；如果风格内容被任何 IDE 或工具改动，脚本会停止。
- 可单独运行 `python3 scripts/verify_style_lock.py` 检查当前风格是否与锁定版本一致。
- 除非用户明确要求修改风格，否则不得改动 `STYLES`、`STYLE_SUFFIX`、风格12/13/15专项规则、构图池和禁用风格集合。
- 如果确实要改风格，必须同步更新 `README.md`、`SKILLA.md`、`references/style-matrix.md`，并重新计算 `STYLE_LOCK_SHA256`。

## 各风格核心规则

### 风格 1 · 写实3DCG画风格
- 哑光珐琅质感（S 45-62%，L 50-65%），表面哑光无镜面反射
- **背景双层结构**：①超大面积同色系柔光漫射（无可见边界，浓雾散射），②极大景深完全模糊的背景元素（仅剩色彩轮廓）
- **补色活力**：主体背光面/暗部边缘施加互补色调（色相差≥120°）柔和环境反射光；屏幕/发光部件带补色温内发光
- 严禁：独立漂浮几何体呈现补色、白色光源、硬阴影、镜面高光

### 风格 2 · 3D立体风格
- 磨砂金属质感（S 30-48%，L 45-60%），微粒漫反射
- 背景同风格1双层结构，色差≤8%
- **补色活力**：主体背面/侧面补色调柔和逆光边缘光（rim light），磨砂面隐含补色光感
- 严禁：白色泛光、多光源叠加、独立漂浮几何元素

### 风格 3 · CG厚涂风格
- S 58-76%，L 52-68%，色彩鲜明不发灰
- **冷暖分色**：受光面主色暖调高光，背光面/阴影边缘渗入互补色冷调（色相差≥120°）
- 强侧硬光、笔触光影、硬边阴影

### 风格 4 · 羊毛毡风格 ⚠️ 仅母婴/儿童
- 高明度柔和色系（S 30-55%，L 70-85%）
- 触发关键词：母婴/儿童/婴幼/幼儿/宝宝/亲子/启蒙

### 风格 5 · 壁画质感风格
- 复古艺术色调（S 52-68%，L 52-65%），饱满有层次，禁止发灰发黑
- **复古光感活力**：天窗丁达尔散射光带补色冷暖偏移，光束本体主色暖调、边缘渗入补色冷调；材质表面交界处出现彩虹薄膜反射光
- 光束边缘必须大范围柔化扩散，严禁清晰硬边光束轮廓线

### 风格 6 · 轻量超现实3D商业插画
- 主色中高饱和（S 50-65%），背景极高明度（L 80-92%），清新通透
- **背景充实**：叠放 1-2 个超大体量相关元素，极深度高斯模糊（仅剩极淡轮廓），色差≤8%
- **补色活力**：主体边缘补色调柔和侧面环境光，屏幕内发光带补色温
- 正面+顶部品牌色有色柔和漫射光，禁止纯白光源

### 风格 7 · 3D极简玻璃拟态
- S 48-68%，L 58-75%，背景同色系极简双色渐变（L≥55%）
- **核心**：主体物直接置于背景上，**严禁任何容器/面板/气泡/玻璃球包裹**；玻璃质感仅体现在主体物自身材质（折射/透明/高光边缘）
- **折射活力**：主体物边缘色散出补色彩虹薄边，焦散光斑散落周围
- 焦散效果、折射光、软阴影、全局照明GI
- ⚠️ **严禁出现任何人物/人形/人体/人脸/手/肢体**，主视觉必须为产品/道具/物件/场景元素

### 风格 8 · 新拟物主义2.0充气3D
- 柔和梦幻渐变（S 52-68%，L 58-72%），背景明亮版（L≥52%），禁止荧光色
- **充气透光活力**：充气体半透明材质，背光侧透出互补色透射光（S 45-60%），边缘呈现冷暖光晕过渡
- 丁达尔效应大范围弥散，无硬边光束
- 严禁：密集缠绕曲线/盘旋流线/复杂螺旋装饰
- ⚠️ **严禁出现任何人物/人形/人体/人脸/手/肢体**，主视觉必须为产品/道具/物件/场景元素

### 风格 9 · 3D粘土C4D潮流玩具
- 主色同色系渐变（S 58-72%，L 55-68%），背景均匀单色（L≥52%）
- **背景**：叠放 1-2 个超大体量物件高斯模糊（色差≤12%），**严禁背景径向渐变光晕/圆形聚光/中心亮斑**
- **粘土光感活力**：粘土主体侧光面/背光面边缘补色轮廓逆光（rim light）
- 单侧45°大柔光漫射，光区均匀扩散

### 风格 10 · 潮流3D艺术混合材质拟物
- S 50-68%，L 55-70%，背景深饱和版（L≥42%，S≥55%），禁止背景发黑发灰
- **混合材质活力**：主体内部自发光本体主色系，外散射时色温偏移向补色，外轮廓/透明材质边缘形成补色发光晕圈；金属玻璃交界面折射产生补色色散
- 柔和全局光+强边缘逆光+内部自发光

### 风格 11 · 2D扁平插画 Rounded Geometric Futurism
- 品牌色约60%（S 52-72%，L 55-72%）+ 辅色20% + 点缀色10%，**全画不超过4色**
- 数字颗粒+半调网点质感，硬边色块阴影
- 主体轮廓局部边缘发光

### 风格 12 · 明亮编辑扁平插画风格
- **配色**：产品主色必须保留为第一识别色，并映射为明亮干净商业色，占画面30-50%；辅色限定为亮黄/金黄、番茄红/珊瑚红、浅粉、浅天蓝、钴蓝、亮草绿/嫩绿、奶油白、少量深绿；整体明亮清爽，不脏不黑，黑色只用于1px纯黑轮廓线和内部细节线，占画面≤5%，不得用于阴影或大面积色块
- **线条**：严格纯2D扁平矢量插画，所有主体、人物、手部、服装、卡片、道具和内部细节线只使用约1px纯黑色连续闭合细描边，线条必须完整包裹形体、转角圆滑、不断裂、不缺口、不毛糙；严禁彩色描边、浅色描边、半透明描边、粗黑轮廓、厚描边、3D边缘光
- **块面**：必须使用清晰硬边封闭平涂色块，颜色边界锐利明确，像SVG矢量路径填色；每个色块内部必须是单一纯色填充，色块之间以明确路径边界区分；严禁渐变、晕染、高光、反光、阴影、模糊、纹理、噪点、半调、透明叠色、白色高光条、刷痕、尾焰、速度残影
- **纯净度**：画面保持完整印刷海报感；所有辅助图形必须是闭合矢量面，颜色区域完整连续，边缘清楚
- **构图**：构图必须大胆但元素克制；信息中心必须安全居中，但镜头和动势不能死板平视居中；优先透视冲刺、中心放射彩带、斜向悬浮平台、极近前景框景、俯视卡片漩涡、低角仰视英雄、局部手势夸张、极简徽章特写；核心主体、人物脸部、关键手势、核心道具和功能识别信息必须完整落在中央70%安全区；画面四边只放可裁剪的纯色背景块、宽大彩带、前景色面或装饰按钮
- **人物（可选）**：只使用现代商业矢量人物或局部手部特写；圆润肢体、自然肤色、现代发型；脸部只用点状眼和短线鼻口，头发为单一深色色块；服装限定为T恤、衬衫、卫衣、裤装或简洁工作服，保持当代商业插画气质
- **道具**：必须图标化，内部细节控制在3处以内；例如麦克风只保留外轮廓和少量按钮，耳机只保留大色块耳罩和头梁
- **辅助图标**：小圆徽章和辅助按钮内部只放单个完整纯色符号：实心圆、星形、爱心、对勾或短线；避免纸张、书页、折角、叠片这类容易破碎的图标
- **输出**：风格12保存为 PNG，不走 200KB JPEG 压缩，并在保存前执行全色块无抖动限色平涂后处理；所有颜色色块都严格禁止纹理、晕染、渐变残留和碎色。该规则作用于风格12的全部填充色块，不针对某一种颜色。

### 风格 13 · 柔体渐变几何插画风格
- 严格纯 2D flat vector illustration，禁止3D渲染、厚度、写实阴影和面分割线
- 主体为抽象几何体/有机块面，使用喷枪渐变、全息折射条、深浅元素层次
- 暗/亮两套色彩模式随机，品牌色保持核心占比，辅助元素有Z轴层次和轻微motion blur
- **专属色相偏移**：主体核心与关键识别面保留品牌主色；背景渐变、辅助几何体和点缀渐变可根据应用名做确定性色相偏移（±60°），用于分散红/蓝大面积背景，不改变主体核心品牌色
- **游戏类、母婴/儿童类禁用**

### 风格 14 · 巨人插画合成风格
- 2D矢量角色叠加真实照片背景，角色尺寸远超正常比例形成超现实反差
- 角色高饱和、极简两面分色、1-2px深色描边；背景照片保持低饱和或自然色
- 可用人/拟人吉祥物/巨大化产品/抽象品牌符号作为巨人主体

### 风格 15 · 绒粒超现实CG风格
- 产品主色必须作为主体物或最大主空间结构的第一主色，占主体/主结构45-65%
- 主体根据关键词选择：建筑/城市/导航/经营等可用圆润构筑物或微型城市；非建筑主题优先巨大化物品、超现实道具、抽象符号、有机雕塑、悬浮孤岛
- 主体核心信息必须居中展示，落在画面中央60%安全区，便于上下左右裁剪适配
- 构图可为巨物近景安全构图、洞口/窗框居中窥视、悬浮平台居中、坡面边缘、室内通道/阶梯、俯视微型岛、特写完整主体、大留白居中孤立物
- 水平地面、多层横向色带、远景不是禁用项，需根据内容和构图判断是否使用，避免无关模板化套用
- 微型角色/小生物按内容生成，豆形小人/坐姿抽象小人/团块生物优先，鸟形低频；全部保持极简圆润和绒粒质感

## 全局约束（所有风格通用）

| 类别 | 规则 |
|------|------|
| **光源色彩** | 严禁中性白/纯白漫射光，必须带品牌色邻近色调的有色色温（暖橙/暖黄/冷青/冷紫） |
| **背景复杂度** | 装饰元素总数不超过3个，严禁密集缠绕曲线/盘旋流线/螺旋形装饰大量包裹主体 |
| **设备规范** | 设备**不是默认视觉元素**，绝大多数画面应以实体物件/符号/场景/人物为主，不出现设备；仅当内容明确涉及操作行为（拍照/导航/直播/视频通话等）时才自然出现；如出现必须使用无品牌特征通用外观，**严禁苹果iPhone/iPad外观/品牌标志** |
| **屏幕UI** | 屏幕内界面极度简化，仅显示1-2个简洁图标/色块/符号，严禁复杂UI/密集图标行/详细截图 |
| **气泡禁止**（风格7除外）| 严禁用气泡/玻璃球/圆形封闭容器/透明球体将主体物包裹在内部 |
| **人物**（风格1-6, 9-10）通用分类 | 优先用产品/道具替代人物；如需人物只能展示背影/侧身/局部（手/手臂/剪影），**严禁正脸/全脸/五官清晰**；严禁真实照片感、严禁卡通大眼萌系 |
| **人物**（风格1-6, 9-10, 11）**游戏类** | 游戏角色/主角可作为次要元素（占画面≤25%），主体以武器/道具/场景/特效为主；角色优先侧身/3/4侧角/背影，如有正脸则人脸占画面≤10%，**严禁大尺寸正脸填充，严禁孤立角色配空洞背景** |
| **人物**（风格7/8）| **完全禁止**，不得出现任何人物/人形/人体/人脸/手/肢体，主视觉必须为产品/道具/物件/场景元素 |
| **儿童类例外** | 母婴/儿童分类下允许迪士尼/皮克斯3D卡通风格（风格1-6, 9-10有效） |
| **人物**（风格11）| 无正脸限制，扁平插画风格 |
| **人物**（风格12）| 明亮编辑扁平插画人物开放：闭合约1px细线描边，圆润肢体、简化五官、自然肤色，可用手部特写或人物互动承载产品叙事；线条必须完整包裹形体 |
| **补色融入方式** | 一律通过物理光现象（rim light/折射/透射/色温偏移）融入，**严禁以独立漂浮几何体/装饰形状呈现** |

| **母婴/儿童风格限制** | 母婴/儿童分类跳过风格 13，风格 4 正常参与 |
| **红色调性控制** | 禁止大面积红色调（红/玫红/橙红/猩红）铺满背景或占画面超过35%；红色系仅用于主体局部点缀和品牌识别；背景和辅助区域优先使用冷色系或中性色中和 |

| **手机壳样式禁止** | 严禁主视觉套用在手机外形/手机壳模板/设备外框内部；画面必须是无边框的完整视觉场景；不得被矩形框/设备轮廓/屏幕外框包围或嵌套 |

```

⑤ 通用兜底：{App名}核心功能视觉符号
```

**过滤规则**：
- **VISUAL_STOPWORDS**：过滤抽象/功能描述词（"软件"、"体验"、"平台"等80+词）
- **LABEL_PATTERNS**：HUMAN_DESC 含机构标签（"安卓"、"官方"、"国家税务"等）时整句丢弃
- **GENERIC_TAGS**：通用分类词从 S_INTRO 提取时跳过（"常用工具"、"其他工具"、"休闲娱乐"等）
- **字符长度过滤**：S_INTRO 提取词仅取2-6字中文词组；HUMAN_DESC ≤20字才作为补充场景氛围，≤24字才作为主要来源

## 点缀色色彩关系（每张随机取一种）

**面积约束**：点缀色总面积严格控制在画面的 **5% 以内**，仅作为光感层次点缀，禁止大面积铺开。

**融入方式**（只能通过光的物理方式，严禁独立漂浮几何体）：
- ① 主体边缘轮廓逆光（rim light）偏向点缀色色温
- ② 发光屏幕/内发光的散射光晕带点缀色温
- ③ 材质折射/玻璃色散/半透明透射产生点缀色光斑
- ④ 冷暖分色中阴影面渗入点缀色调

| 模式 | 色相差 | 示例 |
|------|--------|------|
| 互补色 | 150-180° | 红→青绿，蓝→橙，黄→紫 |
| 对比色 | 120-150° | 红→蓝绿，蓝→黄橙，橙→蓝紫 |
| 撞色 | 90-120° | 红→紫，蓝→黄绿，橙→蓝 |
| 邻近对比色 | 60-90° | 红→黄，蓝→绿，橙→绿 |

## 运行

```bash
# 默认（CSV_PATH 内置 + 前20条 + 输出到桌面 app_covers_style_test）
python3 scripts/batch_generate.py

# 指定 CSV、范围、输出目录
python3 scripts/batch_generate.py \
  --csv "/path/to/data.csv" \
  --start 1 --end 52 \
  --outdir ~/Desktop/your_folder

# 强制使用某个风格（自动跳过游戏类禁用风格）
python3 scripts/batch_generate.py --style 9
```

**命令行参数：**

| 参数 | 默认 | 说明 |
|------|------|------|
| `--csv` | 脚本内 `CSV_PATH` | CSV 文件路径 |
| `--start` | 1 | CSV 起始行号（1-based，含） |
| `--end` | 20 | CSV 结束行号（1-based，含） |
| `--style` | 0 | 强制风格 1-15（0=自动轮换） |
| `--outdir` | `~/Desktop/app_covers_style_test` | 输出目录 |

输出文件名格式：`PACKAGE_NAME.jpg`

## 批次说明

- 自动续接全局风格轮换（受 `--start` 行号影响）
- 游戏类（`PARENT_ID=游戏`）自动跳过风格 7、8、12、13
- 母婴/儿童分类自动跳过风格 13，风格 4 正常参与
- 强制风格命中游戏禁用时自动顺延到下一个合法风格
- 支持 OpenAI 兼容接口及 Replicate、Fal.ai
- 输出 JPEG ≤200KB（PIL 自适应质量压缩）
- **跳过已存在文件**：输出目录中已有同名 `.jpg` 的行自动跳过，支持断点续跑
- **并发生图**：5路并行 API 请求（`MAX_WORKERS=5`），串行预计算风格索引保证轮换连续
- **自动重试**：API 超时/5xx 错误自动重试最多3次（间隔 5 / 15 / 30 秒递增），3次全失败才记录 ❌
